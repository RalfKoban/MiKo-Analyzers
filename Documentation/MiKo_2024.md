# MiKo_2024: Start Enum parameter documentation with default phrase

## Cause

Documentation for an Enum parameter does not begin with a phrase specifying the action to be taken.

## Rule description

The Enum parameter documentation should begin with a phrase specifying the action to be taken with the Enum.
This ensures clarity about how the Enum will be used and what it represents.

### Rationale behind

Enum parameters specify choices or options that affect method behavior.
Their documentation needs to clearly explain what the enum represents and how it affects the method's operation.

Writing clear Enum parameter documentation provides several important benefits:

- **Clear Parameter Purpose**:
  Starting with an action phrase immediately tells developers what the enum parameter controls or specifies.
  This helps them choose the correct enum value for their needs.

- **Better Understanding**:
  When documentation explains what the enum is used for, developers can make informed decisions about which value to pass.
  This leads to more correct method calls and fewer errors.

- **Consistent Pattern**:
  Using a standard phrasing pattern for all enum parameters creates consistency across the codebase.
  Developers know what to expect when reading enum parameter documentation.

- **Framework Alignment**:
  Many .NET Framework methods use similar patterns for enum parameter documentation.
  Following this approach makes custom APIs feel familiar and professional.

- **Reduced Confusion**:
  Clear documentation about what the enum specifies or controls eliminates ambiguity.
  Developers do not need to guess what impact the parameter has on method behavior.

- **Better IntelliSense**:
  When developers see action-oriented enum documentation in IntelliSense, they immediately understand the parameter's role.
  This speeds up development and reduces the need for additional documentation lookups.

## How to fix violations

To fix a violation of this rule, rewrite the enum parameter documentation to start with an action phrase.

Use phrases like `Specifies the...`, `Defines the...`, `Indicates the...`, or `Determines the...` followed by a description of what the enum controls or represents.

## How to suppress violations

```csharp
#pragma warning disable MiKo_2024
#pragma warning restore MiKo_2024
```
