# MiKo_1529: Do not prefix methods with 3rd person singular verb

## Cause

A method name starts with a verb in its third-person singular form, such as `Creates`, `Updates`, or `Calculates`.

## Rule description

Methods should start with a verb in its base form because they represent actions the code performs.
Using the base form (e.g. `Create`, `Update`, `Calculate`) makes method names clear, consistent, and easy to read.
Third-person verbs like `Creates` read like descriptions instead of commands, which can make the intention less direct.

### Rationale behind

Method names that start with a third-person singular verb can be confusing because they sound like descriptions of what something does rather than direct instructions to do something.
This subtle difference makes the intent of the method less clear, especially to developers who are new to the codebase.

Using the base form of verbs for method names provides several important benefits:

- **Clearer Intent**:
  Base-form verbs like `Create`, `Update`, or `Calculate` read as direct commands.
  This makes it immediately clear that calling the method triggers an action.

- **Consistency**:
  Following a uniform naming pattern across all methods makes the codebase easier to navigate.
  Developers can predict what a method does just by reading its name.

- **Alignment with .NET Conventions**:
  The .NET Framework and its ecosystem consistently use the base form of verbs for method names.
  Sticking to this convention makes the code feel natural to anyone familiar with .NET development.

- **Improved Readability**:
  Code that reads like a set of clear instructions is easier to understand at a glance.
  Consistent naming reduces the time needed to understand what a piece of code does.

- **Reduced Cognitive Load**:
  When naming follows a predictable pattern, developers do not have to think about why a name looks different from what they expect.
  This frees up mental capacity for understanding the actual logic of the code.

## How to fix violations

To fix a violation of this rule, rename the method so that its name starts with the base form of the verb instead of the third-person singular form.

## How to suppress violations

```csharp
#pragma warning disable MiKo_1529
#pragma warning restore MiKo_1529
```
