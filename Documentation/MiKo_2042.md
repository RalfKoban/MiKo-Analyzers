# MiKo_2042: Use `<para>` XML tags instead of `<p>` HTML tags in documentation

## Cause

The documentation uses `<p>` HTML tags instead of `<para>` XML tags.

## Rule description

Documentation should use the `<para>` XML tags instead of `<p>` HTML tags.
This ensures a consistent format, suitable for XML-based documentation tools and frameworks.

### Rationale behind

XML documentation comments in .NET are designed to use XML-specific tags, not HTML tags.
While some documentation tools may render HTML tags, using the proper XML tags ensures better compatibility and correctness across all documentation tools and scenarios.

Using `<para>` XML tags instead of HTML `<p>` tags provides several important benefits:

- **Proper XML Structure**:
  Using `<para>` maintains the XML structure of documentation comments.
  This ensures that documentation is well-formed XML that can be properly parsed and validated.

- **Better Tool Compatibility**:
  XML documentation tools and processors are designed to handle `<para>` tags.
  Using the correct XML tags ensures compatibility with all .NET documentation tools.

- **Consistent Documentation Format**:
  Following the XML documentation conventions creates consistency across the codebase.
  This makes documentation more predictable and professional.

- **Improved Processing**:
  Documentation generation tools can process `<para>` tags more reliably than HTML tags.
  This ensures that paragraphs are properly formatted in generated documentation.

- **Standard Compliance**:
  Using `<para>` aligns with the .NET XML documentation standards and best practices.
  This demonstrates adherence to proper documentation conventions.

## How to fix violations

To fix a violation of this rule, replace all `<p>` HTML tags with `<para>` XML tags in the documentation.
Ensure that opening and closing tags are properly matched.

## How to suppress violations

```csharp
#pragma warning disable MiKo_2042
#pragma warning restore MiKo_2042
```
