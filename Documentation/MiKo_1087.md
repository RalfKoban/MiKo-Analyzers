# MiKo_1087: Name constructor parameters after their base class counterparts

## Cause

A constructor parameter does not match the name of the corresponding parameter in the base class constructor.

## Rule description

Constructor parameters should use the same names as the corresponding parameters in the base class constructor.
This consistency prevents confusion and makes it clear how parameters are being passed through the inheritance chain.

### Rationale behind

When a derived class constructor passes parameters to a base class constructor, using different parameter names creates confusion.
Developers have to mentally map between the different names to understand the flow of data.
Using identical parameter names makes the relationship obvious and reduces the chance of errors.

Matching base class parameter names provides several important benefits:

- **Improved Clarity**:
  Identical parameter names immediately show that the parameter is being passed through to the base class.
  Developers do not have to wonder if some transformation or mapping is happening.

- **Reduced Confusion**:
  When parameter names differ, developers must check whether the parameters represent the same thing or different things.
  Matching names eliminate this ambiguity.

- **Easier Maintenance**:
  When base class constructors change, it is easier to update derived classes when the parameter names match.
  Developers can quickly identify which parameters correspond to each other.

- **Better IntelliSense Experience**:
  When calling constructors, seeing consistent parameter names across the inheritance hierarchy helps developers understand the expected values.
  Different names for the same concept create unnecessary mental overhead.

- **Self-Documenting Code**:
  Matching parameter names serve as documentation, showing that the parameter serves the same purpose in both classes.
  This reduces the need for additional comments explaining parameter relationships.

- **Consistent Conventions**:
  Following the base class naming establishes a clear pattern that makes the entire codebase more predictable and easier to work with.

## How to fix violations

To fix a violation of this rule, rename the constructor parameter to match the base class parameter name.

## How to suppress violations

```csharp
#pragma warning disable MiKo_1087
#pragma warning restore MiKo_1087
```
