# MiKo_6072: Surround base class calls with blank lines

## Cause

Base class calls are not surrounded with blank lines.

## Rule description

Surround calls to the base class with blank lines to visually separate them from other code.
This improves readability and makes the code easier to follow.

### Rationale behind

Surrounding calls to the base class with blank lines is important for keeping code clear and well-organized.
A base class call, such as `base.DoSomething();`, delegates behavior to the parent class.
This is a distinct concern from the rest of the method body, and visually separating it helps developers immediately understand where inherited behavior is invoked.

Surrounding base class calls with blank lines provides several important benefits:

- **Clear Delegation of Behavior**:
  Blank lines make it obvious that a call is forwarding work to the parent class.
  Developers can quickly identify the boundary between inherited behavior and the logic added in the overriding method.

- **Improved Code Readability**:
  Visual separation makes the code flow easier to follow.
  The blank lines act as natural breaks that guide the reader through the structure of the method.

- **Better Code Organization**:
  Separating base class calls from surrounding statements helps organize the method into distinct sections.
  Each section can be understood independently without having to trace through unrelated lines.

- **Easier Maintenance**:
  When base class calls are clearly separated, modifying the method is simpler.
  It is immediately clear which part of the code relies on the parent class and which part adds new behavior.

- **Reduced Cognitive Load**:
  Visual separation reduces the mental effort required to understand the method's structure.
  Developers do not need to analyze every line in detail to find where inherited behavior is invoked.

- **Consistent Code Style**:
  Consistently surrounding base class calls with blank lines establishes a predictable pattern across the codebase.
  This makes code reviews and onboarding easier.

## How to fix violations

To fix a violation of this rule, add blank lines before and after base class calls.
Ensure each base class call is visually separated from the surrounding code.

## How to suppress violations

```csharp
#pragma warning disable MiKo_6072
#pragma warning restore MiKo_6072
```
