# MiKo_3080: Use `switch ... return` instead of `switch ... break` for assignments

## Cause

A switch statement assigns values to a variable, field, property, or parameter using `break` statements instead of returning values directly.

## Rule description

Using switch statements to assign values within methods increases complexity and reduces readability.
Simplify this by moving the switch statement into its own method.
Let each section of the switch return values directly, instead of assigning them to a shared variable, field, property, or parameter and using break statements.
This makes the code easier to read and change later.

### Rationale behind

Switch statements that assign to a variable, field, property, or parameter and use `break` create unnecessary complexity and make the code harder to follow.
When a local variable is used, it must be declared just to hold the result, assigned in each case, and then returned at the end - all extra steps for something that could be done directly.
When a property or field is assigned instead, no new variable is needed, but each case changes it as a side effect, which makes it harder to see what value it will have at the end.

Using direct returns from switch statements provides several important benefits:

- **Reduces Code Complexity**:
  Removing the break statements (and any extra variable used just to hold the result) makes the switch statement simpler.
  Each case clearly shows what value is produced without having to track how a variable, property, or field changes over time.

- **Improves Readability**:
  Direct returns make it immediately obvious what value each case produces.
  There is no need to scan through the code to find where the variable is declared or returned, or to figure out which case last changed a property or field.

- **Prevents Assignment Errors**:
  When using a shared variable, field, property, or parameter, it is possible to forget to assign it in some cases or to assign it multiple times.
  Direct returns eliminate these errors because each case must explicitly produce a value.

- **Enables Expression-Based Patterns**:
  Modern C# supports switch expressions that are even shorter than switch statements.
  Moving to direct returns prepares the code for that.

- **Simplifies Method Structure**:
  Moving the switch into its own method with direct returns creates a focused method that does one thing.
  This makes the code easier to understand and test.

- **Makes Control Flow Clearer**:
  It is immediately clear that each case ends there - no code runs after it.
  Break statements can be confused with breaking out of loops or can be accidentally omitted.

## How to fix violations

To fix a violation of this rule, move the switch statement into its own method.
Change each case to return the value directly instead of assigning to a variable, field, property, or parameter and using `break`.
Call this new method from the original location and use its return value.

## How to suppress violations

```csharp
#pragma warning disable MiKo_3080
#pragma warning restore MiKo_3080
```
