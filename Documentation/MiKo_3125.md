# MiKo_3125: NUnit tests should not have both [Test] and [TestCase] applied

## Cause

A test method is marked with both `[Test]` and `[TestCase]` at the same time.

## Rule description

Avoid marking a test method with both [Test] and [TestCase] at the same time.

While [Test] marks a simple, non-parameterized test and [TestCase] marks a parameterized test, NUnit allows combining them without generating extra test cases.
Use either [Test] or [TestCase] to keep tests clear and easy to understand.

### Rationale behind

`[Test]` and `[TestCase]` serve different purposes in NUnit.
`[Test]` marks a simple test without parameters, while `[TestCase]` marks a parameterized test that provides specific input values to run the test multiple times.
When both attributes are applied to the same method, NUnit allows it but does not generate extra test cases.
This mix of attributes creates confusion about the intent of the test.

This rule provides several important benefits:

- **Clearer Test Intent**:
  A test method with only one attribute immediately communicates its purpose.
  Mixing both attributes makes it unclear whether the test is meant to be simple or parameterized.

- **Improved Readability**:
  Tests with a single, well-chosen attribute are easier to read and understand at a glance.
  Developers do not need to check for both attributes to determine the test type.

- **Reduced Confusion**:
  Having both `[Test]` and `[TestCase]` on the same method can mislead developers into thinking the test runs multiple times, which it does not.
  This creates a false understanding of the test behavior.

- **Consistent Test Structure**:
  A codebase where tests consistently use either `[Test]` or `[TestCase]` is easier to navigate.
  Mixing both attributes makes it harder to quickly identify which tests are simple and which are parameterized.

- **Easier Maintenance**:
  When refactoring or extending tests, a single attribute makes the intended test type clear.
  A test with both attributes requires more thought to decide how to change it correctly.

## How to fix violations

To fix a violation of this rule, remove the `[Test]` attribute from test methods that already have `[TestCase]` attributes applied.

## How to suppress violations

```csharp
#pragma warning disable MiKo_3125
#pragma warning restore MiKo_3125
```
