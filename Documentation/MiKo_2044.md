# MiKo_2044: Reference method parameters correctly in documentation

## Cause

Method parameters are not referenced correctly in the documentation.

## Rule description

Method parameters should be referenced using `<paramref name="..."/>` in the documentation, not `<param name="..."/>` or plain text.
This allows the documentation to update automatically if the parameter is renamed or deleted.

### Rationale behind

Referencing parameters correctly in documentation ensures that the documentation remains accurate and synchronized with the code.
When parameters are properly referenced, refactoring tools can automatically update the documentation when parameter names change.

Using `<paramref name="..."/>` for parameter references provides several important benefits:

- **Automatic Refactoring Support**:
  When parameters are renamed using refactoring tools, `<paramref>` references are automatically updated.
  This keeps documentation synchronized with the code without manual intervention.

- **Compile-Time Validation**:
  Some documentation tools can validate that `<paramref>` references point to actual parameters.
  This helps catch documentation errors during the build process.

- **Consistent Formatting**:
  Parameters referenced with `<paramref>` are consistently styled in documentation viewers and IntelliSense.
  This makes parameter references stand out and easier to identify.

- **Better Readability**:
  Properly tagged parameter references are visually distinct from regular text.
  This helps developers quickly identify which parts of the documentation refer to parameters.

- **Reduced Maintenance**:
  Using `<paramref>` reduces the effort needed to maintain documentation when parameters change.
  The documentation stays accurate with less manual work.

- **Semantic Correctness**:
  Using the correct tag provides semantic meaning to the documentation.
  Tools can understand that the reference is to a parameter, not just formatted text.

## How to fix violations

To fix a violation of this rule, replace plain text parameter references or incorrect tags with `<paramref name="..."/>` where the name attribute matches the actual parameter name.
Ensure all parameter references throughout the documentation use this tag.

## How to suppress violations

```csharp
#pragma warning disable MiKo_2044
#pragma warning restore MiKo_2044
```
