# MiKo_2216: Use `<paramref>` instead of `<param>` to reference parameters

## Cause

The documentation uses `<param>` tags to reference parameters outside of the parameter documentation section.

## Rule description

The `<param>` tag is for describing parameters, whereas the `<paramref>` tag is used to indicate that a word in the code comments refers to a parameter.
When referencing a parameter in sections like `<remarks>`, use the `<paramref>` tag to maintain clarity and proper documentation standards.

### Rationale behind

XML documentation tags have specific purposes, and using them correctly ensures that documentation tools can process and format them appropriately.
The `<param>` tag is designed for the parameter documentation section where each parameter is described.

The `<paramref>` tag is designed to create references to parameters from within other documentation sections.
It allows documentation tools to format parameter references differently, often with special styling or linking, making them easy to identify.

Following this rule provides several important benefits:

- **Proper Tool Processing**:
  Documentation tools can correctly identify and format parameter references when the right tag is used.

- **Better Visual Formatting**:
  `<paramref>` references often appear with special styling in IntelliSense and generated documentation.

- **Clearer References**:
  Using `<paramref>` makes it immediately obvious that text refers to a parameter name.

- **Semantic Correctness**:
  Using tags according to their intended purpose creates semantically correct documentation.

- **Professional Standards**:
  Following XML documentation conventions demonstrates knowledge of best practices.

## How to fix violations

To fix a violation of this rule, replace `<param>` tags used for referencing parameters with `<paramref>` tags in sections like `<summary>` or `<remarks>`.

## How to suppress violations

```csharp
#pragma warning disable MiKo_2216
#pragma warning restore MiKo_2216
```
