# MiKo_1524: Do not prefix methods with 'Sub_'

## Cause

A method name starts with the `Sub_` prefix.

## Rule description

Avoid naming methods with the 'Sub_' prefix. This prefix does not add useful information because any method can call any other method. Use clear and descriptive method names instead.

### Rationale behind

Using `Sub_` as a prefix is a form of notation akin to Hungarian notation.
It attempts to indicate that a method is a "sub-routine" of another method.
However, any method can be invoked by any other method.
Therefore, the prefix conveys no reliable semantic or design information, adds noise, and reduces readability.

Avoiding such prefixes and using clean, intention-revealing names instead provides several important benefits:

- **Improved Readability**:
  Method names without artificial prefixes are easier to read and understand at a glance.
  A well-chosen name immediately communicates what the method does, without requiring the reader to mentally strip away a meaningless prefix first.

- **Better Maintainability**:
  When method names clearly express their purpose, it is easier to locate relevant code when a bug needs to be fixed or a feature needs to be changed.
  Names that carry no real meaning make it harder to navigate a codebase over time.

- **Reduced Cognitive Load**:
  Prefixes like `Sub_` force the reader to process extra noise before understanding what the method actually does.
  Removing such prefixes means less mental effort is needed to read and understand the code.

- **Consistency with Established Conventions**:
  The .NET naming guidelines do not use Hungarian-style prefixes for method names.
  Following these conventions makes the code feel familiar to other .NET developers and easier to work with.

- **Clearer Intent**:
  A method name should describe what the method does, not how it relates to the call hierarchy.
  Call relationships change as code evolves, so embedding them in names creates names that quickly become misleading.

## How to fix violations

To fix a violation of this rule, rename the method by removing the `Sub_` prefix and choosing a name that clearly describes what the method does.

## How to suppress violations

```csharp
#pragma warning disable MiKo_1524
#pragma warning restore MiKo_1524
```
