# MiKo_6062: Indent values in element initializers correctly

## Cause

Values inside an element initializer are placed on a new line but not at the correct indentation.

## Rule description

Code readability improves when values inside element initializers are placed consistently, either on the same line as the opening brace or indented to that position on a new line.
This makes the code clearer and easier to follow.

### Rationale behind

Inconsistent indentation makes it harder to read what an element initializer contains.
When each value starts at the expected position, the structure of the initializer is immediately clear.

Following a consistent indentation for element initializers provides several important benefits:

- **Improved Readability**:
  When all values inside an element initializer start at a consistent column, the structure is immediately visible.
  This makes it easier to scan the list of values at a glance.

- **Easier Code Reviews**:
  Consistent indentation makes it straightforward to compare element initializers when reviewing changes in version control.
  Formatting differences do not distract from the actual logical changes being reviewed.

- **Reduced Cognitive Load**:
  Developers do not need to mentally adjust to varying indentation when reading through code.
  Code that follows a predictable pattern requires less effort to understand.

- **Cleaner Visual Alignment**:
  Proper indentation ensures that all values align vertically in multi-line element initializers.
  This visual grouping makes it easier to recognize which values belong together and to notice any that are missing.

- **Better Consistency Across the Codebase**:
  When all element initializers follow the same indentation rule, the code feels uniform.
  This uniformity also makes it easier for developers new to the codebase to learn and follow its conventions.

## How to fix violations

To fix a violation of this rule, place the values at the same position they would occupy in the single-line form of the element initializer.
That means right after the opening brace `{` followed by a space.

## How to suppress violations

```csharp
#pragma warning disable MiKo_6062
#pragma warning restore MiKo_6062
```
