# MiKo_6062: Align expressions within complex initializer expressions relative to opening brace

## Cause

An expression within a complex initializer expression is not at the expected indentation position relative to the opening brace `{`.

## Rule description

Code readability improves when expressions within complex initializer expressions are placed consistently relative to the opening brace.
This makes the code clearer and easier to follow.

### Rationale behind

Inconsistent indentation of expressions within complex initializer expressions makes it harder to quickly read and understand what is being created.
When each expression is indented at the expected position relative to the opening brace, the structure of the initializer is immediately clear.

Following a consistent indentation rule for complex initializer expressions provides several important benefits:

- **Improved Readability**:
  When all expressions within a complex initializer start at a consistent column, the structure of the initializer is immediately visible.
  This makes it easier to scan the list of values at a glance.

- **Easier Code Reviews**:
  Consistent indentation makes it straightforward to compare complex initializers when reviewing changes in version control.
  Formatting differences do not distract from the actual logical changes being reviewed.

- **Reduced Cognitive Load**:
  Developers do not need to mentally adjust to varying indentation when reading through code.
  Code that follows a predictable pattern requires less effort to understand.

- **Cleaner Visual Alignment**:
  Proper indentation ensures that all expressions align vertically in multi-line complex initializers.
  This visual grouping makes it easier to recognize which values belong together and to notice any that are missing.

- **Better Consistency Across the Codebase**:
  When all complex initializers follow the same indentation rule, the code feels uniform.
  This uniformity also makes it easier for developers new to the codebase to learn and follow its conventions.

## How to fix violations

To fix a violation of this rule, adjust the indentation of the first expression so that it starts at the expected position relative to the opening brace.
For multi-line complex initializers, ensure the first expression is indented by exactly one indentation level from the column of the opening brace.

## How to suppress violations

```csharp
#pragma warning disable MiKo_6062
#pragma warning restore MiKo_6062
```
