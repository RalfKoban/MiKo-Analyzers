# MiKo_2046: Reference type parameters correctly in documentation

## Cause

Type parameters are not referenced correctly in the documentation.

## Rule description

The XML documentation should reference type parameters properly via `<typeparamref name="..." />` instead of plain text or incorrect tags.

### Rationale behind

Generic type parameters need to be referenced correctly in documentation to ensure that the documentation remains accurate and synchronized with the code.
When type parameters are properly referenced, refactoring tools can automatically update the documentation when type parameter names change.

Using `<typeparamref name="..." />` for type parameter references provides several important benefits:

- **Automatic Refactoring Support**:
  When type parameters are renamed using refactoring tools, `<typeparamref>` references are automatically updated.
  This keeps documentation synchronized with the code without manual intervention.

- **Compile-Time Validation**:
  Documentation tools can validate that `<typeparamref>` references point to actual type parameters.
  This helps catch documentation errors during the build process.

- **Consistent Formatting**:
  Type parameters referenced with `<typeparamref>` are consistently styled in documentation viewers and IntelliSense.
  This makes type parameter references stand out and easier to identify.

- **Better Readability**:
  Properly tagged type parameter references are visually distinct from regular text.
  This helps developers quickly identify which parts of the documentation refer to type parameters.

- **Reduced Maintenance**:
  Using `<typeparamref>` reduces the effort needed to maintain documentation when type parameters change.
  The documentation stays accurate with less manual work.

- **Semantic Correctness**:
  Using the correct tag provides semantic meaning to the documentation.
  Tools can understand that the reference is to a type parameter, not just formatted text.

## How to fix violations

To fix a violation of this rule, replace plain text type parameter references or incorrect tags with `<typeparamref name="..." />` where the name attribute matches the actual type parameter name.
Ensure all type parameter references throughout the documentation use this tag.

## How to suppress violations

```csharp
#pragma warning disable MiKo_2046
#pragma warning restore MiKo_2046
```
