# MiKo_6036: Align lambda blocks with the corresponding arrow(s)

## Cause

A lambda block is on a different line than the corresponding arrow (`=>`) but not placed directly below it.

## Rule description

Lambda blocks are easier to distinguish from other code blocks when placed either on the same line as the arrow or directly below it.
This makes the code clearer and easier to follow.

### Rationale behind

Lambda expressions are common in modern C# code.
When a lambda block is not aligned with its arrow, it can be hard to visually distinguish the lambda body from other surrounding code blocks such as method bodies or conditional statements.

Aligning lambda blocks with their corresponding arrows provides several important benefits:

- **Improved Visual Clarity**:
  The connection between the arrow and the lambda body is immediately obvious.
  Lambda expressions stand out from surrounding code and are easier to identify at a glance.

- **Better Code Organization**:
  Consistent alignment creates a clear visual hierarchy.
  The structure of the code becomes easier to follow, especially when multiple lambda expressions appear near each other.

- **Reduced Confusion**:
  A lambda block that is aligned with the arrow is less likely to be confused with other code blocks such as method bodies or conditional statements.
  This is especially helpful in code with nested lambdas.

- **Faster Code Comprehension**:
  Code that follows a consistent pattern is easier to scan and understand quickly.
  Identifying lambda expressions becomes straightforward when their blocks are always aligned with the arrow.

- **Consistent Code Style**:
  Applying the same alignment pattern for all lambda blocks creates a uniform codebase.
  The same visual pattern appears throughout the code, which reduces the mental effort required to read different parts of it.

## How to fix violations

To fix a violation of this rule, place the lambda block either on the same line as the arrow or directly below it on the next line.

## How to suppress violations

```csharp
#pragma warning disable MiKo_6036
#pragma warning restore MiKo_6036
```
