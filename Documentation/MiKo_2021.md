# MiKo_2021: Start parameter documentation with default phrase

## Cause

Parameter documentation does not follow the standard phrasing conventions.

## Rule description

The documentation of parameters should describe what they represent.

### Rationale behind

Parameter documentation helps developers understand what values to pass to methods and what those values represent.
Clear, well-structured parameter documentation is essential for API usability and reduces the likelihood of incorrect usage.

Writing effective parameter documentation provides several important benefits:

- **Clear Parameter Purpose**:
  Good documentation explains what each parameter represents and how it affects the method's behavior.
  Developers can call methods correctly without examining the implementation.

- **Reduced Errors**:
  When developers understand what parameters expect, they are less likely to pass incorrect values.
  This leads to fewer runtime errors and bugs.

- **Better IntelliSense Support**:
  Parameter documentation appears directly in IntelliSense when developers type method calls.
  This provides immediate guidance without needing to look at separate documentation.

- **Improved API Usability**:
  Well-documented parameters make APIs easier to learn and use.
  Developers can be productive more quickly without needing to ask questions or search for examples.

- **Consistent Documentation Style**:
  Following standard phrasing conventions creates consistency across all methods in the codebase.
  This makes the documentation more predictable and easier to understand.

- **Professional Standards**:
  Clear parameter documentation demonstrates attention to quality and user experience.
  This makes the API appear more professional and well-designed.

## How to fix violations

To fix a violation of this rule, rewrite the parameter documentation to clearly describe what the parameter represents.

Use descriptive phrases that explain the parameter's purpose, valid values, and any constraints or special considerations.

## How to suppress violations

```csharp
#pragma warning disable MiKo_2021
#pragma warning restore MiKo_2021
```
