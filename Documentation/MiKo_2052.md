# MiKo_2052: Use default phrase for `ArgumentNullException` documentation

## Cause

An `ArgumentNullException` documentation does not use the default phrase.

## Rule description

When documenting an `ArgumentNullException`, use the specific phrase: `<paramref name="value"/> is <see langword="null"/>.`
This approach ensures clarity and consistency, helping developers understand the condition under which the exception is thrown.

### Rationale behind

`ArgumentNullException` is one of the most common exceptions in .NET, thrown when a null argument is passed to a method that does not accept null values.
Using a consistent phrase for documenting this exception makes the documentation predictable and immediately clear.

Using the default phrase for `ArgumentNullException` documentation provides several important benefits:

- **Consistent Exception Documentation**:
  Using the same phrase across the codebase creates uniformity in how `ArgumentNullException` is documented.
  Developers know exactly what to expect when they see this exception type.

- **Aligned with .NET Framework**:
  The .NET Framework uses similar phrasing for `ArgumentNullException` documentation, and following the same pattern makes the codebase feel more professional and aligned with .NET conventions.

- **Immediate Clarity**:
  The standard phrase immediately tells developers which parameter cannot be null.
  There is no ambiguity about the exception condition.

- **Reduced Documentation Errors**:
  Following a standard template reduces the likelihood of documentation errors or unclear wording.
  The documentation is correct by default.

- **Better Tool Support**:
  Code analysis tools and documentation generators can better recognize and validate properly formatted `ArgumentNullException` documentation.
  This improves overall code quality.

- **Faster Comprehension**:
  When all `ArgumentNullException` documentation follows the same pattern, developers can quickly scan through method documentation and identify null checks.
  This saves time and improves productivity.

## How to fix violations

To fix a violation of this rule, update the `ArgumentNullException` documentation to use the phrase `<paramref name="value"/> is <see langword="null"/>.` where `value` is replaced with the actual parameter name.
Ensure the parameter reference and null keyword use the appropriate XML tags.

## How to suppress violations

```csharp
#pragma warning disable MiKo_2052
#pragma warning restore MiKo_2052
```
